## Complexity Ordering Rationale

Each category orders use cases by complexity based on:

- **Integration requirements** — More systems = more complexity
- **Clinical risk** — Higher stakes require more validation
- **Real-time needs** — Latency constraints add difficulty
- **Regulatory exposure** — FDA/compliance adds overhead
- **Data challenges** — Quality, volume, privacy constraints
- **Operational maturity required** — Change management and workflow integration
